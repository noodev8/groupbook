# Side Product Build Framework

**Total Budget: 36 Hours**

---

## Phase 0: Requirements & Scope Finalisation
**Time: 4 hours**

- Clarify the exact problem being solved
- Define core user roles and flows
- Decide what is in and explicitly out of scope
- Lock success criteria for "done"

> **Goal:** No ambiguity, no scope creep, no second-guessing later.

---

## Phase 1: Architecture & Project Setup
**Time: 2 hours**

- Define core entities and relationships
- Data model and permissions
- Project scaffolding (repo, environments, config)

> **Goal:** Stable foundation with no architectural drift.

---

## Phase 2: Core Functionality
**Time: 14 hours**

- End-to-end happy path
- Create / edit / delete flows
- Business rules enforced
- Edge cases handled
- Core reporting logic (raw, not polished)

> **Goal:** The product works reliably without explanation.

---

## Phase 3: UX Polish & Responsiveness
**Time: 10 hours**

- Mobile-first responsive layouts
- Clear spacing, typography, hierarchy
- Button states, loading states, errors
- Sensible defaults and guardrails
- Visual calm and consistency

> **Goal:** It feels obvious and usable for a real business.

---

## Phase 4: Deployment & Release Setup
**Time: 2 hours**

- Production deployment
- Certificates / domains
- Environment variables
- Smoke testing live system

> **Goal:** It runs safely in the real world.

---

## Phase 5: Monetisation & Final Checks
**Time: 4 hours**

- Stripe subscription (single plan)
- Feature gating (free vs paid)
- Upgrade prompts
- End-to-end sanity checks

> **Goal:** Money works, nothing breaks.

---

## Completion Rule (Non-Negotiable)

At 36 hours:

- [x] The core problem is solved
- [x] UX is calm and presentable
- [x] Product is live
- [x] Payments work

**No further work unless real users ask.**
